FIX 2 — Stock restoration only on FULL refund
═══════════════════════════════════════════════════════════════
The current logic restores ALL invoice items to stock even on a partial
refund. That is wrong: on a partial money refund, the customer keeps
the items physically; double-restoring would let the shop sell them again.

Required behavior:
- FULL refund   (newPaidAmount === 0)  →  restore all items to stock as today.
- PARTIAL refund (newPaidAmount  >  0) →  do NOT restore any stock. Items
                                          remain with the customer.

Implementation:
- The computation `const newPaidAmount = invoice.paid_amount - totalRefund;`
  and `const newStatus = newPaidAmount === 0 ? 'returned' : 'partially_returned';`
  already exists. Hoist these BEFORE building stmts (they may already be
  positioned correctly — verify).
- Wrap the existing stock-restoration loop in a guard:

    if (newStatus === 'returned') {
      for (const item of items) {
        if (!item.product_id) continue;
        stmts.push({
          sql: `UPDATE products SET stock_qty = stock_qty + ?, updated_at = ?
                WHERE id = ? AND track_stock = 1`,
          params: [item.quantity, now, item.product_id],
        });
      }
    }

- Do not change the items loop's intent — only gate it by the full-refund
  condition.
